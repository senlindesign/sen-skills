# i-write — 个人写作系统

让 AI 写出你亲笔写的文章。

原则一句话:**先懂人,再写字。**

## 一篇公开文章是什么做的

把任何一篇公开写作拆开,里面有四样东西:

1. **观点(frame)** — 文章背后,作者「非说不可」的那个判断。它从作者的价值观和看世界的视角里长出来 → `frame.md`
2. **笔触(style)** — 同一个观点,一百个人写出一百种声音:句子的节奏、用词的癖好、开头和收尾的习惯 → `style.md`
3. **技巧(craft)** — 观点怎么拆解、怎么铺排、怎么落刀。这是公共的手艺,不专属于谁 → `craft.md`
4. **媒介(medium)** — 同一篇文章,放在公众号和小红书上呈现不同:长度、排版、钩子、标签 → `medium.md`

前两样是你的,谁也拿不走;后两样是公共的工具和规则。这个 skill 做的事,就是把四样东西装回一篇文章里。

## 工作方式

**第一次使用:Deep Interview**(`interview.md`)

写字之前先认识你。访谈挖两样东西:

- 你的价值观、信仰、看世界的视角——不靠你自我总结,靠跨领域提问和作品回溯「测」出来 → 生成 `profile/frame.md`
- 你已有的写作偏好、句式、审美与禁忌——你的笔触 → 生成 `profile/style.md`

题量三档可选:短(20 题)、中(50 题)、长(100 题)。

**之后每次写作:**

```
你口述想说的
   ↓
frame.md   帮你把观点站稳(主流怎么说 → 你的空位在哪)
   ↓
 确认观点骨架(你点头才动笔)
   ↓
style.md   给文章装上你的笔触
   ↓
craft.md   工具箱优化写法(AI 痕迹过滤默认必做,其余你选)
   ↓
medium.md   按媒介规则出各版本(清单外的媒介会先调研)
   ↓
成文,每个媒介一份
```

## 文件地图

```
i-write/
├── SKILL.md          执行流程(AI 读的操作手册)
├── interview.md      Deep Interview 指南
├── craft.md          写作工具箱
├── medium.md         媒介规则(公众号/小红书/即刻/LinkedIn/X/Medium.com/Substack)
└── profile/          你的部分——访谈生成
    ├── frame.md      视角:你怎么看
    └── style.md      笔触:你怎么写
```

## 常用指令

- 「帮我写一篇 XX」— 进写作流程(没有 profile 会先访谈)
- 「i-write」— 直接唤起 skill
- 「重新访谈」「更新我的风格」— 增量更新 profile
- 「这篇发小红书和公众号」— 多媒介版本
